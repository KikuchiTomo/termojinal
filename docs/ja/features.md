## GPU アクセラレーテッドレンダリング

Termojinal は macOS 上の Metal バックエンドを使う **wgpu** で全描画を GPU 上で行う。高リフレッシュレートや大きなウィンドウサイズでも滑らかで低遅延な描画を実現する。

### 仕組み

1. **フォントアトラス** — fontdue を使って CPU 上でグリフをラスタライズし、単一のテクスチャアトラスにパッキングして GPU に一度だけアップロードする。
2. **インスタンスレンダリング** — 表示中の各セルをインスタンス化されたクワッドとして1回のドローコールで描画し、アトラス内の対応するグリフを参照する。
3. **WGSL シェーダー** — xterm 256色パレット全体とトゥルーカラー（24ビット RGB）に対応するカスタム頂点・フラグメントシェーダー。
4. **SDF レンダリング** — コマンドパレットなどの UI 要素には、Gaussian ブラーによるフロストガラス効果を持つ SDF（Signed Distance Field）角丸矩形を使用。
5. **画像パイプライン** — インライン画像（Kitty, Sixel, iTerm2）は個別の GPU テクスチャとしてアップロードされ、テキストと合成される。

### 体感

- 大量の出力をスクロールしても滑らか。
- ウィンドウのリサイズが即座に反映される — 空白のフラッシュなし。
- 半透明ウィンドウやブラーエフェクトがフルフレームレートで動作。

## Daemon-Owned PTY アーキテクチャ

Termojinal は **Daemon-owned PTY** モデルを採用している。GUI（`Termojinal.app`）はシンクライアントであり、デーモン（`termojinald`）に Unix ソケット経由でバイナリフレーミングプロトコル（JSON + バイナリフレーム）を使って接続する。

### これが意味すること

- **セッションが GUI の再起動を生き残る** — GUI を閉じて再度開いても、シェルはそのまま動作し続ける。
- `tm exit` — GUI をデーモンから切断する。すべてのシェルは動作し続ける。
- `tm kill` — デーモン内のシェルセッションを終了する。

### セキュリティ

- ソケットディレクトリのパーミッション: `0o700`
- ソケットファイルのパーミッション: `0o600`
- `read_line` は 1 MB 制限を適用してメモリ枯渇を防止
- シェルパス許可リスト検証により任意のバイナリの実行を防止

## ワークスペース

ワークスペースは単一の Termojinal ウィンドウ内の独立した環境。各ワークスペースは独自のタブとペインを持ち、作業ディレクトリも独立している。

| 操作 | ショートカット |
|------|---------------|
| 新規ワークスペース | `Cmd+N` |
| ワークスペース N に切替 | `Cmd+1` 〜 `Cmd+9` |
| 次のワークスペース | `Cmd+Shift+]` |
| 前のワークスペース | `Cmd+Shift+[` |

ワークスペースはサイドバーに名前、git ブランチ、アクティブなポート、AI エージェントのステータスとともに表示される。アクティブなワークスペースはハイライトされる。

**ユースケース:** ワークスペース 1 で開発サーバー、ワークスペース 2 で Claude Code、ワークスペース 3 でデータベースツールを実行し、`Cmd+1`/`2`/`3` で瞬時に切り替える。

---

## タブ

各ワークスペースは複数のタブを持てる。タブはワークスペースのサイドバーエントリを共有するが、ターミナルセッションは独立している。

| 操作 | ショートカット |
|------|---------------|
| 新規タブ | `Cmd+T` |
| タブを閉じる | `Cmd+W` |
| 次のタブ | `Cmd+Shift+}` |
| 前のタブ | `Cmd+Shift+{` |

### タブタイトルのフォーマット

タブバーはフォールバックチェーン形式の文字列をサポートする:

```
{title|cwd_base|Tab {index}}
```

- `title` — シェルが設定したタブタイトル（OSC エスケープシーケンス経由）
- `cwd_base` — カレントディレクトリのベース名
- `Tab {index}` — 1始まりのタブインデックスを含むリテラルテキスト

最初に空でない値が使われる。`config.toml` の `[tab_bar]` でカスタマイズ可能。

### タブバーのオプション

- タブが1つだけの場合の表示/非表示（`always_show`）
- 位置: 上部または下部
- カラー、高さ、タブの最小/最大幅のカスタマイズ
- アクティブタブの下線アクセント

## 分割ペイン

現在のペインを水平または垂直に分割して、1つのタブ内で複数のターミナルセッションを並べて表示できる。

| 操作 | ショートカット |
|------|---------------|
| 右に分割（水平） | `Cmd+D` |
| 下に分割（垂直） | `Cmd+Shift+D` |
| 次のペイン | `Cmd+]` |
| 前のペイン | `Cmd+[` |
| ペインのズーム（フルスクリーン切替） | `Cmd+Shift+Enter` |

### イミュータブルツリーレイアウト

ペイン分割は**イミュータブル分割ペインツリー**（関数型の永続データ構造）で管理される。これにより:

- ペイン構成が堅牢かつ予測可能。
- 分割を任意の深さでネスト可能。
- 各分割はセパレーターのドラッグで調整できる比率を保持。

### ペインセパレーター

- セパレーターの色と幅がカスタマイズ可能。
- ドラッグ可能 — つかんでドラッグしてペインをリサイズ。
- ヒット許容範囲が設定可能（`separator_tolerance`）で、1〜2px の幅でもつかみやすい。

### フォーカスボーダー

フォーカス中のペインには色付きのボーダーが表示される（アルファ透過対応）。色と幅はカスタマイズ可能。

### スクロールバー

各ペインは独自のスクロールバーを持ち、サムとトラックの不透明度がカスタマイズ可能。

### タブドラッグによるペイン分割

タブをペインエリアにドラッグすることでレイアウトを分割できる。左・右・上・下のエッジにドロップして、そのタブの内容で新しい分割を作成する。

### ペインをタブに抽出

`Cmd+Shift+T` でフォーカス中のペインを新しいタブとして抽出する。分割ペインをフルタブに昇格させたい場合に便利。

## Claudes Dashboard

`Cmd+Shift+C` で開く。全ワークスペースの Claude Code セッションを管理する lazygit 風の2ペインインターフェース。

### レイアウト

- **左ペイン** — ステータスインジケーター付きの全 Claude Code セッション一覧
- **右ペイン** — 選択したセッションの詳細と最近のアクティビティ

### 表示内容

- セッションステータス（アクティブ、アイドル、権限待ち）
- 各セッションが実行されているワークスペースとタブ
- 権限リクエスト履歴
- クイックアクション: セッションへジャンプ、保留中リクエストの承認/拒否

## Quick Launch

`Cmd+O` で開く。ワークスペース・タブ・ペインを素早く切り替えるためのファジー検索オーバーレイ。

### 機能

- 全ワークスペース名、タブタイトル、ペインコンテンツに対するファジーマッチング
- キーボード駆動: 入力でフィルタリング、矢印キーで選択、Enter でジャンプ
- 結果にワークスペース/タブ/ペインの階層構造を表示

## サイドバー

ウィンドウ左側の縦方向のサイドバーがナビゲーションのハブになっている。

切替: `Cmd+B`

### 表示内容

各ワークスペースについて:

- **ワークスペース名** とカラードット
- **Git ブランチ** 名（例: `main`, `feature/login`）
- **Git 変更状態** インジケーター（未コミットの変更がある場合に黄色のドット）
- **アクティブなポート** セッション内で検出されたもの
- **通知ドット** （オレンジ）未読アラート用
- **Allow Flow インジケーター** — 保留中の AI 権限リクエストがあるワークスペースの左側にアクセントストライプ
- **AI エージェントステータス** — Claude Code（または他の AI エージェント）セッションがアクティブかどうかを表示。インジケータースタイルは設定可能

### エージェントステータスインジケーター（Hooks ベース）

Claude Code のステータスは **Claude Code Hooks** を通じてイベント駆動で検出される（ポーリングなし）。ワークスペースで Claude Code セッションが検出されると、サイドバーにそのステータスが表示される:

| スタイル | 挙動 |
|---------|------|
| `"pulse"` | ワークスペースのドットがアニメーションの光で脈動する（デフォルト） |
| `"color"` | ドットが固定色に変わる（紫 = アクティブ、黄 = 権限待ち） |
| `"none"` | エージェントインジケーターなし |

### サイドバーのリサイズ

サイドバーの幅はドラッグ可能。右端をつかんでドラッグしてリサイズ:

- デフォルト幅: 240px
- 最小幅: 120px
- 最大幅: 400px

## コマンドパレット

`Cmd+Shift+P` で開く。ビルトインアクションとカスタムコマンドにアクセスするためのファジー検索オーバーレイ。

### ビジュアルデザイン

パレットは GPU 上で完全にレンダリングされる:

- **SDF 角丸矩形** 任意の解像度でくっきりした角
- **フロストガラスブラー** パレット背後のぼかし（ブラー半径のカスタマイズ可能、デフォルト 20px）
- **ドロップシャドウ** 半径と不透明度がカスタマイズ可能
- **暗いオーバーレイ** パレット背後でターミナルコンテンツを暗くする

### 機能

- ファジーテキストマッチング — コマンド名の一部を入力してフィルタリング
- キーボードナビゲーション — 矢印キーで選択、Enter で実行、Escape で閉じる
- スクロール — `max_visible_items`（デフォルト 10）を超えるとリストがスクロール
- 幅はウィンドウの割合で設定可能（`width_ratio`、デフォルト 60%）

### コマンドの種類

| 種類 | 表示 | 説明 |
|------|------|------|
| Builtin | "Builtin" | Termojinal に同梱されたアクション |
| Plugin | "Plugin" | 署名なしのユーザー作成コマンド |
| PluginVerified | チェックマーク付き "Verified" | 有効な Ed25519 署名で署名されたコマンド |

## Allow Flow

Allow Flow は Termojinal の AI エージェント連携システム。Claude Code などのツールがファイル編集やシェルコマンド実行、リソースへのアクセスの権限を必要とする場合、Termojinal がリクエストをインターセプトし、フォーカスを切り替えることなく承認・拒否できるようにする。

### 仕組み

1. Claude Code が構造化された IPC 経由で `PermissionRequest` フックを送信する（ターミナル出力のパースではない）。
2. Termojinal が**通知**と対象ペイン下部の**ヒントバー**を表示。
3. **サイドバー**で保留中のリクエストがあるワークスペースに青いアクセントストライプが表示される。
4. どのワークスペースからでも1つのキーで応答:

| キー | 動作 |
|------|------|
| `y` | このリクエストを1件許可 |
| `n` | このリクエストを1件拒否 |
| `Y` | このワークスペースの保留中リクエストをすべて許可 |
| `N` | このワークスペースの保留中リクエストをすべて拒否 |
| `a` / `A` | 許可して**ルールを記憶**（永続ルールを作成） |
| `Esc` | ヒントバーを閉じる |

### カスタム検出パターン

ビルトインの Claude Code 連携に加え、任意のツールの権限プロンプトを検出する正規表現パターンを追加できる:

```toml
[[allow_flow.patterns]]
tool = "My Deploy Tool"
action = "production deploy"
pattern = "Deploy to production\\? \\[y/N\\]"
yes_response = "y\n"
no_response = "n\n"
```

### 設定オプション

| オプション | デフォルト | 説明 |
|-----------|-----------|------|
| `overlay_enabled` | `true` | ペイン内にヒントバーを表示 |
| `side_panel_enabled` | `true` | サイドバーに保留中リクエストを表示 |
| `auto_focus` | `false` | リクエスト到着時にペインを自動フォーカス |
| `sound` | `false` | 権限リクエスト時にサウンドを再生 |

## Quick Terminal

画面端からスライドするQuake風ドロップダウンターミナル。グローバルホットキーで操作でき、Termojinal がフォーカスされていなくても動作する。

**デフォルトホットキー:** `Cmd+`` `（デーモンの起動が必要）

### 仕組み

Mac のどこからでもホットキーを押すと、画面上部からターミナルウィンドウがスライドダウンする。もう一度ホットキーを押す（または Escape）で閉じる。Quick Terminal は専用のワークスペースを持つため、セッションはトグル間で維持される。

### 設定

| オプション | デフォルト | 説明 |
|-----------|-----------|------|
| `enabled` | `true` | 機能の有効化 |
| `hotkey` | `"cmd+\`"` | グローバルホットキー（カスタマイズ可能） |
| `animation` | `"slide_down"` | アニメーション: `slide_down`, `slide_up`, `fade`, `none` |
| `animation_duration_ms` | `200` | アニメーション速度 |
| `height_ratio` | `0.4` | 画面に対する高さの割合（0.0〜1.0） |
| `width_ratio` | `1.0` | 画面に対する幅の割合（0.0〜1.0） |
| `position` | `"center"` | 水平位置: `left`, `center`, `right` |
| `screen_edge` | `"top"` | スライド元のエッジ: `top`, `bottom` |
| `hide_on_focus_loss` | `false` | フォーカスを失った時に自動非表示 |
| `dismiss_on_esc` | `true` | Escape で非表示 |
| `show_sidebar` | `false` | Quick Terminal でサイドバーを表示 |
| `show_tab_bar` | `false` | Quick Terminal でタブバーを表示 |
| `show_status_bar` | `true` | Quick Terminal でステータスバーを表示 |
| `window_level` | `"floating"` | スタッキング: `normal`, `floating`, `above_all` |
| `corner_radius` | `12.0` | 角丸の半径（ピクセル） |
| `own_workspace` | `true` | Quick Terminal 専用ワークスペース |

## ディレクトリツリー

サイドバー内のファイルブラウザパネルで、カレントディレクトリのディレクトリ構造を表示する。

### 有効化

`config.toml` で有効にする:

```toml
[directory_tree]
enabled = true
```

### 機能

- **Git 対応ルート検出** — ツリーのルートを以下に設定可能:
  - `auto` — git リポジトリ内ならリポジトリルートを使用、そうでなければ CWD
  - `cwd` — 常にカレントディレクトリを使用
  - `git_root` — 常に git リポジトリルートを使用
- **展開/折りたたみ可能なディレクトリ** — クリックまたはキーボードで展開
- **色分けされたエントリ** — ディレクトリとファイルで異なる色（カスタマイズ可能）
- **キーボードナビゲーション** — 矢印キーで移動、プレフィックス検索でジャンプ
- **ファイル操作:**
  - ディレクトリをダブルクリックで `cd`
  - ファイル上で `v` を押すとエディタで開く（`$EDITOR` またはデフォルトの nvim）
- **スクロール** — `max_visible_lines`（デフォルト 20）を超えるとツリーがスクロール

## Time Travel

Time Travel は OSC 133 シェル統合マーカーを使ってコマンド履歴を追跡し、コマンド間のナビゲーションや以前のセッションの復元を可能にする。

### コマンド履歴

- **記録** — OSC 133 エスケープシーケンス（zsh, bash, fish など現代的なシェルがサポート）により、コマンド境界が自動的に追跡される。
- **ナビゲーション** — `Cmd+Up` / `Cmd+Down` でコマンド間をジャンプ。
- **コマンドマーカー** — 左ガターに各コマンドの開始位置を示すビジュアルマーカー。
- **コマンド位置** — ステータスバーに現在位置を表示（例: "Command 15/42"）。
- **容量** — セッションあたり最大 10,000 件のコマンドレコードを保存（設定可能）。

### タイムライン UI

`Cmd+Shift+T` で開く。コマンド履歴のビジュアルタイムラインで素早くナビゲーション。

### セッションの永続化

- **終了時に保存** — Termojinal 終了時にセッション状態全体が保存される。
- **起動時に復元** — Termojinal 起動時に前回のセッションが自動的に復元される。
- **名前付きスナップショット** — セッションの特定の時点に名前を付けて保存。セッションあたり最大 50 スナップショット（設定可能）。

### 設定

```toml
[time_travel]
command_history = true           # コマンド記録の有効化
max_command_history = 10000      # セッションあたりの最大レコード数
command_navigation = true        # Cmd+Up/Down ナビゲーション
show_command_marker = true       # ガターマーカー
show_command_position = true     # ステータスバーの位置表示
timeline_ui = true               # Cmd+Shift+T タイムライン
session_persistence = true       # 終了時に保存
restore_on_startup = true        # 起動時に復元
snapshots = true                 # 名前付きスナップショット
max_snapshots_per_session = 50   # 最大スナップショット数
```

## ステータスバー

ウィンドウ下部のカスタマイズ可能なバーで、セッションのリアルタイム情報を表示する。

### セグメント

ステータスバーは**左揃え**と**右揃え**のセグメントで構成される。各セグメントは以下を持つ:

- `content` — `{variable}` プレースホルダーを含むテンプレート文字列
- `fg` — 前景（テキスト）色
- `bg` — 背景色

### 利用可能なテンプレート変数

| 変数 | 説明 |
|------|------|
| `{user}` | 現在のユーザー名 |
| `{host}` | システムのホスト名 |
| `{cwd}` | フルパスの作業ディレクトリ |
| `{cwd_short}` | ホームを `~` に置換した作業ディレクトリ |
| `{git_branch}` | 現在の git ブランチ |
| `{git_status}` | git ステータスの要約 |
| `{git_remote}` | git リモート名 |
| `{git_worktree}` | git worktree パス |
| `{git_stash}` | stash の数 |
| `{git_ahead}` | リモートより先行しているコミット数 |
| `{git_behind}` | リモートより遅れているコミット数 |
| `{git_dirty}` | 変更されたファイル数 |
| `{git_untracked}` | 未追跡ファイル数 |
| `{ports}` | セッション内のリッスンポート |
| `{shell}` | シェル名（zsh, bash など） |
| `{pid}` | シェルのプロセス ID |
| `{pane_size}` | ペインのサイズ（列 x 行） |
| `{font_size}` | 現在のフォントサイズ |
| `{workspace}` | ワークスペース名 |
| `{workspace_index}` | ワークスペースインデックス（1始まり） |
| `{tab}` | タブ名 |
| `{tab_index}` | タブインデックス（1始まり） |
| `{time}` | 現在時刻（HH:MM） |
| `{date}` | 現在日付（YYYY-MM-DD） |

### 例

```toml
[status_bar]
enabled = true

[[status_bar.left]]
content = "{user}@{host}"
fg = "#1A1A28"
bg = "#7AA2F7"

[[status_bar.left]]
content = "{cwd_short}"
fg = "#C0C0CC"
bg = "#1A1A28"

[[status_bar.left]]
content = "{git_branch} +{git_ahead} -{git_behind}"
fg = "#A6E3A1"
bg = "#0F0F18"

[[status_bar.right]]
content = "{time}"
fg = "#1A1A28"
bg = "#7AA2F7"
```

## インライン画像

Termojinal は3つのプロトコルを使ってターミナル内で直接画像を表示できる。

### サポートプロトコル

| プロトコル | 説明 |
|-----------|------|
| **Kitty Graphics Protocol** | APC ベースのプロトコル。RGB, RGBA, PNG をサポート。ダイレクト送信、チャンク転送、配置、削除に対応。 |
| **iTerm2 Inline Images** | iTerm2 や他のターミナルで使われる OSC 1337 プロトコル。 |
| **Sixel Graphics** | DCS ベースのレガシーなターミナルグラフィックスフォーマット。 |

### 仕組み

画像データはエスケープシーケンス経由で受信、デコードされ、個別の GPU テクスチャとしてアップロードされる。レンダリング時にテキストコンテンツと合成されるため、画像コンテンツのスムーズなスクロールとリサイズが可能。

## CJK・日本語入力

Termojinal は日本語入力をファーストクラスの要件として設計されている。

### CJK 全角文字

- 2カラムを占める文字（CJK 漢字、全角カタカナなど）は倍幅でレンダリングされる。
- 曖昧幅の文字も正しく処理される。

### インライン IME（Input Method Editor）

- macOS の IME 変換中の状態がカーソル位置にインラインでレンダリングされる。
- 変換中のテキストは専用の背景色（`preedit_bg`）で表示されるため、入力中の内容が一目でわかる。
- 別の入力ウィンドウもコンテキスト切り替えも不要 — ターミナル内で直接日本語を入力できる。

---

## カラー絵文字

カラー絵文字は macOS の Core Text を使ってレンダリングされ、他のネイティブ macOS アプリケーションと同じ絵文字表示になる。テキストと一緒に正しい幅でインライン表示される。

## テーマと外観

### ビルトインテーマ

デフォルトテーマは **Catppuccin Mocha** パレットに準拠 — 目に優しい暖かみのあるダークテーマ。

### カスタムテーマ

`~/.config/termojinal/themes/<name>.toml` にテーマファイルを作成する。テーマファイルは `config.toml` の `[theme]` セクションと同じ構造:

```toml
# ~/.config/termojinal/themes/nord.toml
background = "#2E3440"
foreground = "#D8DEE9"
cursor = "#D8DEE9"
selection_bg = "#434C5E"
black = "#3B4252"
red = "#BF616A"
green = "#A3BE8C"
# ... (ANSI 16色すべて)
```

### ダーク/ライトの自動切替

macOS のダーク/ライトモード切り替え時に自動でテーマを切り替えられる:

```toml
[theme]
auto_switch = true
dark = "catppuccin-mocha"
light = "catppuccin-latte"
```

### 外観オプション

| オプション | デフォルト | 説明 |
|-----------|-----------|------|
| ウィンドウの不透明度 | `1.0` | 半透明ウィンドウ（0.0〜1.0） |
| 太字の明るさ | `1.2` | 太字テキストの明るさ倍率 |
| 薄暗い不透明度 | `0.6` | 薄い（faint）テキストの不透明度 |
| フォントサイズズーム | `Cmd+=` / `Cmd+-` | フォントサイズの調整（`max_size` まで） |

## キーバインド

すべてのキーバインドは `~/.config/termojinal/keybindings.toml` で完全にカスタマイズ可能。

### 3レイヤーシステム

| レイヤー | アクティブになる条件 |
|---------|---------------------|
| **normal** | Termojinal がフォーカスされている、通常のシェル実行中 |
| **global** | Termojinal がフォーカスされていなくても常にアクティブ（macOS の CGEventTap 経由） |
| **alternate_screen** | TUI アプリ（nvim, htop など）が代替スクリーンモードで実行中 |

ユーザーのオーバーライドはデフォルトに**マージ**される — 変更したいバインドだけ指定すればよい。

### デフォルトキーバインド

| キー | アクション |
|------|-----------|
| `Cmd+D` | 右に分割 |
| `Cmd+Shift+D` | 下に分割 |
| `Cmd+Shift+Enter` | ペインのズーム |
| `Cmd+]` / `Cmd+[` | 次/前のペイン |
| `Cmd+T` | 新規タブ |
| `Cmd+W` | タブを閉じる |
| `Cmd+Shift+}` / `{` | 次/前のタブ |
| `Cmd+N` | 新規ワークスペース |
| `Cmd+1`〜`Cmd+9` | ワークスペース N に切替 |
| `Cmd+Shift+]` / `[` | 次/前のワークスペース |
| `Cmd+Shift+P` | コマンドパレット |
| `Cmd+O` | Quick Launch |
| `Cmd+Shift+C` | Claudes Dashboard |
| `Cmd+B` | サイドバー切替 |
| `Cmd+F` | 検索 |
| `Cmd+=` / `Cmd+-` | フォントサイズ拡大/縮小 |
| `Cmd+Shift+T` | ペインをタブに抽出 |
| `Cmd+K` | スクロールバッククリア |
| `Cmd+L` | 画面クリア |
| `Cmd+C` / `Cmd+V` | コピー/ペースト |
| `Cmd+,` | 設定を開く |
| `Cmd+Q` | 終了 |
| `Cmd+`` ` | Quick Terminal の切替（グローバル） |
| Option+クリック | URL をブラウザで開く / パスを `open` で開く |

### 特殊アクション

| アクション | 説明 |
|-----------|------|
| `passthrough` | キーを PTY に直接転送（Termojinal をバイパス） |
| `none` | デフォルトバインドを無効化 |
| `{ "command" = "name" }` | カスタムコマンドまたはプラグインを実行 |

### オーバーライドの例

```toml
[normal]
"cmd+d" = "new_tab"                          # Cmd+D をリマップ
"cmd+q" = "none"                             # 誤終了を防止
"cmd+shift+r" = { "command" = "start-review" }  # カスタムコマンド

[alternate_screen]
"cmd+c" = "passthrough"                      # nvim に Cmd+C を渡す
```

## 通知

デスクトップ通知は macOS の **NSNotificationCenter**（`UNUserNotificationCenter` API）経由で配信される。

### 通知が送信されるタイミング

- **コマンド完了** — 長時間実行コマンドが終了した時
- **Allow Flow リクエスト** — AI エージェントが権限を必要とする時
- **カスタムコマンド** — コマンドが `notify` フィールド付きの `done` メッセージを送信した時

### 設定

```toml
[notifications]
enabled = true    # 通知の有効/無効
sound = true      # 通知時にサウンドを再生（デフォルト）
```

## カスタムコマンド

カスタムコマンドはコマンドパレットを通じて Termojinal を拡張するスクリプト。**stdin/stdout 上の行区切り JSON** で通信し、任意の言語（Bash, Python, Deno, コンパイル済みバイナリ）で記述できる。

### ディレクトリ構造

```
~/.config/termojinal/commands/
├── my-command/
│   ├── command.toml    # マニフェスト
│   └── run.sh          # エントリポイント
```

### インタラクティブ UI の種類

コマンドは以下の UI 要素を提示できる:

| タイプ | 説明 |
|--------|------|
| `fuzzy` | フィルタリング可能な単一選択リスト |
| `multi` | チェックボックス付きのフィルタリング可能な複数選択リスト |
| `confirm` | はい/いいえダイアログ |
| `text` | オプションの補完候補付きテキスト入力フィールド |
| `info` | 進捗メッセージ（ファイアアンドフォーゲット） |
| `done` | 完了シグナル（オプションの通知付き） |
| `error` | エラーメッセージ |

### バンドルコマンド

| コマンド | 説明 |
|---------|------|
| `hello-world` | プロトコルのデモ |
| `start-review` | GitHub PR レビューワークフロー（`gh` CLI が必要） |
| `switch-worktree` | Git worktree の切替 |
| `kill-merged` | マージ済みブランチのクリーンアップ |
| `clone-and-open` | リポジトリのクローンとオープン |
| `run-agent` | AI エージェントの起動（Claude Code, Codex, Aider） |

### SDK

型付きの **Deno SDK**（`@termojinal/sdk`）が提供されており、高レベルヘルパーが利用可能: `fuzzy()`, `multi()`, `confirm()`, `text()`, `info()`, `done()`, `error()`。

プロトコルの詳細は [command.md](command.md) を参照。

## MCP サーバー

MCP（Model Context Protocol）サーバーは Claude Code に Termojinal ワークスペースの直接制御を提供する。

### バイナリ

`termojinal-mcp` — stdio 上の JSON-RPC 2.0 で通信し、Unix ソケット経由で Termojinal に接続する。

### 機能

- タブの作成と管理
- ターミナルコンテンツの読み取り
- Allow Flow 権限リクエストの承認・拒否
- サイドバーの AI エージェントステータスの更新

### セットアップ

Claude Code の MCP 設定に追加:

```json
{
  "mcpServers": {
    "termojinal": {
      "command": "termojinal-mcp"
    }
  }
}
```

## セッションデーモン

セッションデーモン（`termojinald`）はバックグラウンドで実行され、**すべての PTY セッションを所有する**。GUI はシンクライアントであり、Unix ソケット上のバイナリフレーミングプロトコルで接続する。

### 機能

- **PTY 所有** — デーモンがすべての擬似端末セッションを所有する。GUI の再起動をシェルが生き残る。
- **バイナリフレーミングプロトコル** — 効率的な GUI-デーモン通信のため、Unix ソケット上で JSON + バイナリフレームを使用。
- **グローバルホットキー** — macOS の CGEventTap 経由でシステム全体のキーボードイベントをインターセプト。Termojinal がフォーカスされていなくても Quick Terminal のホットキーが動作する。
- **セッション永続化** — アプリの再起動時にセッション状態を保存・復元。

### 要件

- **アクセシビリティ権限** — CGEventTap の動作にはシステム設定でのアクセシビリティ権限が必要。

### デーモンの起動

```bash
# Homebrew 経由
brew services start termojinal

# launchctl 経由
launchctl load ~/Library/LaunchAgents/com.termojinal.daemon.plist
```

## コマンド署名

コマンドは **Ed25519** で暗号署名して信頼性を確立できる。

### 仕組み

1. 鍵ペアの生成: `termojinal-sign --generate-key`
2. コマンドの署名: `termojinal-sign path/to/command.toml <secret-key-hex>`
3. 検証はロード時に自動的に行われる。

### 信頼レベル

| ステータス | パレット表示 | 説明 |
|-----------|-------------|------|
| 未署名 | "Plugin" | 署名なし（ユーザー作成コマンドのデフォルト） |
| 検証済み | チェックマーク付き "Verified" | 署名が公式の公開鍵と一致 |
| 不正 | 警告インジケーター | 署名はあるが検証に失敗 |

## Git 連携

Termojinal は git リポジトリの情報を自動的に検出し、UI 全体に表示する。

### 表示される情報

| 場所 | 情報 |
|------|------|
| サイドバー | ブランチ名、変更状態インジケーター |
| ステータスバー | ブランチ、リモート、stash 数、ahead/behind、変更ファイル数、未追跡ファイル数、worktree |
| ディレクトリツリー | Git 対応ルート検出 |

Git 情報は作業に応じて自動的に更新される。

## ポート検出

Termojinal はターミナルセッション内のリッスンポートを自動検出し、サイドバーとステータスバーの `{ports}` テンプレート変数で表示する。`lsof` や `netstat` を実行せずに、どの開発サーバーが動作しているかを確認するのに便利。

## カラー付きコピー
